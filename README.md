# QuatSystem v5.0 — AI-Powered Quantitative Crypto Trading Bot

QuatSystem is a modern, modular, and AI-assisted quantitative trading framework specifically built for **crypto markets**, deeply integrated with the **CoinSwitch Pro API**. It features precise real-time market data ingestion, dynamic regime filtering, strict risk management, and a beautiful Streamlit-based dashboard complete with an AI assistant.

---

## 🚀 Key Features

* **Multi-Strategy Core:** Uses dynamic trend and regime detection (Ranging, Trending Bull, Trending Bear, Volatile breakout) using indicators like RSI, ADX, Bollinger Bands, and EMAs.
* **Live Trading & Paper Trading:** Seamlessly switch between testing your strategies virtually or executing live capital trades.
* **Strict Affordability & Risk Gates:** Automatically scales position sizes based on a set percentage of your available capital, preventing over-leveraging and respecting exchange minimum order values (e.g. ₹100).
* **Live Dashboard:** Monitor PnL, active trades, rejected signals, and the bot's live log directly from your browser.
* **AI Copilot (`QuatAI`):** A conversational AI inside the dashboard to query your performance, adjust settings, and get market insights.
* **Data Cleansing:** Prevents "NO DATA" bottlenecks by adjusting for real-world exchange spreads.

---

## ⚙️ How It Works (The Engine)

The core logic operates in a continuous loop powered by `Bot._run_iteration()`.

1. **Market Scanning:** Every `TRADING_INTERVAL_MINUTES` (usually 1 minute), the bot fetches 5m and 15m OHLCV candles for all tracked pairs from CoinSwitch.
2. **Signal Generation (`signal_engine.py`):** Calculates technical indicators to deduce the current market regime. 
3. **Strict Validation (`asset_filter.py`):** Filters out pairs with low liquidity or abnormally wide spreads (to protect against slippage on low-volume Indian INR markets).
4. **AI Veto (`ai_engine.py`):** (Optional) The underlying signals can be passed to the Gemini-based AI to veto or confirm trades based on complex pattern logic.
5. **Execution (`bot.py`):** If a signal survives all filters, it calculates the proper position size and sends a Market Order.
6. **Position Management:** Once in a trade, it continually checks active positions against their programmed Take-Profit (TP) or Stop-Loss (SL) targets.

---

## 💸 How It Places Orders (Execution Logic)

Execution is handled by `exchange.py` acting strictly on the CoinSwitch API documentation.

1. **Validation & Sizing:** When the Signal Engine outputs a `BUY` or `SELL` signal, the Bot computes the `position_value` (e.g. Risking 2% of the total ₹475 balance).
2. **Type Casting:** Numpy floats generated by Pandas indicators are safely stripped back to native Python `float()`, ensuring JSON serializability.
3. **Payload Construction:** As per CoinSwitch API design:
   * It builds an authentication payload using your API keys and an Ed25519 signature.
   * Modifies quantities and prices back into `str()` types specifically for the JSON payload.
   * Bypasses the `price` parameter entirely for **Market Orders**, ensuring they execute immediately at current market depth.
4. **Placement:** Fires an HTTP POST request to `POST /trade/api/v2/order`.
5. **Logging:** Records the successful Order ID into the local SQLite database (`trades.db`).

## 🛠️ Setup Instructions

### 1. Prerequisites
- Python 3.10+
- `pip install -r requirements.txt` (including `streamlit`, `pandas`, `requests`, `google-genai`, etc.)
- A **CoinSwitch Pro** account with API Keys.

### 2. Environment Configuration
Create a `.env` file in the root directory (use `env.example` as a template). Example:

```env
# API Keys
COIN_SWITCH_API_KEY=your_api_key
COIN_SWITCH_SECRET_KEY=your_secret_key
GEMINI_API_KEY=your_google_ai_key

# Trading Configuration
PAPER_TRADING=false
INITIAL_CAPITAL=475.85
MAX_SPREAD_PCT=1.5
MIN_ORDER_VALUE_INR=100

# Email Alerts
SMTP_EMAIL=your_email@gmail.com
SMTP_PASS=your_google_app_password
```

*(Note: For Gmail notifications, you must use an [App Password](https://myaccount.google.com/apppasswords), NOT your main Google password).*

### 3. Running the System

You must start two processes simultaneously.

**Terminal 1: Start the Trading Bot Engine**
```bash
# This continuously scans the market and places trades
python -m core.bot
```

**Terminal 2: Start the Web Dashboard**
```bash
# This hosts your visual UI and AI Chat
streamlit run app.py
```

---

## ⚠️ Warning

If `PAPER_TRADING=false`, this system will execute **live financial transactions** using your loaded exchange balance. Ensure your risk parameters (like `MAX_RISK_PER_TRADE`) are set conservatively until you trust the bot's behavior in live markets.
